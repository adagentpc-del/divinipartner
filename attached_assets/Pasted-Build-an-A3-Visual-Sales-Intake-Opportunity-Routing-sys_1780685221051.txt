- Company Name

- Website
- Industry
- Account Owner: Alyssa / Drew / Retta
- Account Status: Active / Prospect / Past Client / Lost / Dormant
- Notes
- Uploads
- Last Contact Date
- Next Follow-Up Date

Important:
When a client submits an intake form, the system should check if the company already exists in the Accounts database.

If company name matches an existing account:
- Automatically route the submission to the assigned account owner
- Do not ask the client to choose the sales rep
- Show internal note: “Matched existing account owner”

If company does not exist:
- Use the intake form link source to assign ownership
- Example:
  - /intake/alyssa assigns Alyssa
  - /intake/drew assigns Drew
  - /intake/retta assigns Retta

If no rep source exists:
- Route to Super Admin review queue

- Upload areas

---

# IMPORTANT BUILD RULES

Do not remove existing Partner Portal features.

Add this as a new module inside the portal.

Use Supabase auth and database tables.

Create proper role-based access control.

Protect sales rep data so reps only see their own records.

Super Admin can see everything.

Make sure form submissions create both:
1. Intake submission record
2. Opportunity record

Make account matching case-insensitive and tolerant of punctuation differences.

Example:
“Home Depot”
“The Home Depot”
“Home Depot Inc.”
should be recognized as likely matches or flagged for review.

Start by building the database schema, auth roles, pages, routing logic, and forms.
