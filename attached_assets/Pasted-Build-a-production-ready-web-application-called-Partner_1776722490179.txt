Build a production ready web application called Partner Commerce Portal.

This system is a multi partner ordering, package selection, venue branding, inventory, and fulfillment portal. It must be designed for speed, simplicity, and scale. The interface should feel premium, modern, clean, and highly intuitive. The user experience must minimize typing and maximize clickable selections, dropdowns, toggles, cards, steppers, saved presets, autofill behavior, and reusable templates.

The system is not an A3 only portal. It must support multiple suppliers and vendors, with A3 as one supplier option. The platform owner controls intake, approvals, pricing visibility, supplier assignment, and financial workflow.

Primary users:
1. Super Admin
2. Internal Admin / Project Manager
3. Partner Manager
4. Client Ordering User
5. Vendor / Supplier Fulfillment User

There are two distinct portal types that must both be supported:

Portal Type 1: Client Controlled Ordering Portal
Use case example: Social Commerce Festival
This portal supports:
- multiple cities
- multiple events
- city specific venue setup
- package tiers
- add on items
- hardware inventory by city
- print only vs full hardware logic
- shipping details
- venue contact details
- event based ordering
- partner manager login to create and manage events, cities, and venue logistics

Portal Type 2: Branding Partner Portal
Use case examples: Move Miami and Wynwood Marketplace
This portal supports:
- preloaded venue branding zones
- preloaded products and install areas
- client selects branding areas visually or from cards
- uploads artwork
- enters event and billing details
- submits and pays
- simplified ordering flow with minimal setup by the client

Core product goal:
Build the app skeleton, UI, data structure, workflows, and admin system now, with quote attachments and backend catalog support built in from the beginning.
