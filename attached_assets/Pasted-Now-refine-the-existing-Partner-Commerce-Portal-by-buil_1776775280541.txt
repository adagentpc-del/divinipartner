- partner created
- partner launched live
- onboarding started
- onboarding completed
- first event created
- first order submitted
- first supplier assignment completed
- first invoice created
- first asset approved
- first vendor update submitted
- first reconciliation completed

Also track ongoing behaviors such as:
- number of events created per partner
- number of orders submitted per partner
- number of active users by role
- time to first order
- time to first successful workflow completion
- repeated use of templates/duplication
- usage of bulk tools
- usage of exports
- usage of analytics dashboards

Requirements:
- build clean internal adoption dashboards
- allow filtering by partner, portal type, role, and date range
- do not build invasive tracking
- keep metrics practical and product-ops focused

-----------------------------------
2. WORKFLOW FRICTION INTELLIGENCE
-----------------------------------

Build a friction analysis layer that helps identify where users get stuck or slowed down.

Track and surface signals such as:
- abandoned onboarding steps
- abandoned event creation
- abandoned order flows
- repeated edit/reopen behavior
- repeated validation errors on the same step
- assets repeatedly missing at the same line item stage
- repeated supplier reassignment
- repeated billing corrections
- repeated reconciliation discrepancies
- long delays between workflow stages

Create internal views showing:
- most abandoned flows
- most common error points
- longest time-in-stage workflows
- workflows with most manual overrides
- workflows with highest issue/exception rates

Keep this practical and operational, not vanity product metrics.

-----------------------------------

-----------------------------------

Identify and improve the most frequently used workflows.

Focus on tightening and simplifying:
- create event
- duplicate event
- submit order
- upload assets
- map assets to line items
- assign supplier
- update supplier status
- create invoice
- verify commission
- reconcile order
- approve artwork
- mark item production-ready

Requirements:
- reduce clicks where possible
- reduce visual clutter
- improve button placement
- improve table row actions
- add keyboard-friendly shortcuts if feasible
- add faster repeat patterns
- improve mobile responsiveness for quick actions if relevant
- make the “next best action” more obvious

Do not redesign everything.
Refine based on real operational use.

-----------------------------------
4. REAL-WORLD FORM AND FIELD REDUCTION
-----------------------------------

Reduce unnecessary fields and simplify data entry where practical.

Requirements:
- identify fields rarely used or repeatedly skipped
- convert manual text inputs into smarter defaults/selectors where possible
- collapse advanced/internal-only fields
- create “essential vs advanced” sections on dense forms
- reduce repeated entry across order, event, and billing flows
- improve autofill and inheritance from partner/event defaults
- improve copy-from-existing behavior
- save common contacts, addresses, and settings more aggressively where useful

The goal is to make the product feel lighter without losing operational control.

5. FASTER REPEAT-ORDER AND REPEAT-EVENT FLOWS
-----------------------------------

Build stronger reuse tools for recurring business.

Support fast actions such as:
- reorder from prior order
- duplicate event with smarter defaults
- repeat venue setup
- reorder same package setup
- reuse prior supplier routing
- reuse prior asset mappings where appropriate
- reuse prior billing setup
- reuse prior shipping/install notes

Requirements:
- make these flows feel safe and obvious
- allow selective reuse of parts of a past workflow
- warn when reused data may be outdated
- significantly reduce setup time for repeat partners and repeat events

This is especially important for multi-city recurring event partners.
