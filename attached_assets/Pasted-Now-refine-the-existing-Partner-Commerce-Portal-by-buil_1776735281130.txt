Primary objective for this pass:

1. Asset architecture and file intelligence
2. Asset-to-order and asset-to-line-item mapping

-----------------------------------

Support asset records for:
- artwork files
- proof files
- spec/reference files
- photos
- supporting documents
- install diagrams
- shipping documents
- invoices or internal-only docs if already supported

For each asset record, store:
- title
- file
- file type
- mime type
- uploaded by
- upload date
- linked object type
  - partner
  - event
  - order
  - order line item
  - product
  - package
  - branding zone
  - supplier packet
- linked object id
- asset category
  - client artwork
  - approved artwork
  - proof
  - print-ready file
  - reference
  - install reference
  - shipping document

  - internal only
  - partner visible
  - client visible

- version number

- approval status
- production readiness status
- notes
- tags if feasible

Do not treat files as random uploads.

-----------------------------------
2. ASSET TO ORDER AND LINE ITEM MAPPING
-----------------------------------

Build clear mapping between assets and fulfillment targets.

Requirements:
- assets can attach at order level
- assets can attach at line-item level
- assets can attach to specific products, packages, and branding zones
- line items should clearly show which assets are mapped to them
- one asset may map to multiple line items where appropriate
- one line item may require multiple assets

Examples:
- one event logo file used across multiple ordered products
- one canopy artwork file mapped to one line item
- one zone proof mapped to one branding zone
- one installation diagram attached to a supplier-facing packet

UI requirements:
- line-item asset panel
- drag/drop or select existing asset to map
- clear “mapped assets” display
- clear “missing required asset” display
- ability to remap or replace assets

-----------------------------------
3. ARTWORK VERSIONING AND APPROVAL WORKFLOW
-----------------------------------

Build a practical artwork version and approval workflow.

For artwork assets, support statuses such as:
- uploaded
- under review
- revision requested
- approved
- superseded
- vendor released
- archived

Requirements:
- keep version history visible
- mark one version as current
- mark one version as approved for production
- allow replacing prior files without losing history
- show who approved and when if feasible
- allow internal admin to request revision
- allow internal admin to release approved asset to vendor visibility
- show warning if a vendor is seeing a non-approved asset

Important:
A product or line item should not appear production-ready unless its required assets are approved or intentionally overridden.

4. FILE COMPLETENESS AND PRODUCTION READINESS CHECKS
-----------------------------------

Build lightweight but real readiness checks.

For each line item or zone, support flags such as:
- artwork missing
- artwork uploaded
- artwork mapped
- proof required
- proof approved
- print-ready file present
- dimensions confirmed
- attachment method confirmed
- spec reference attached
- vendor ready
- production blocked

Use product capability rules and zone configuration where possible to determine expected asset needs.

Examples:
- graphic-only item may require artwork + dimensions confirmation
- custom zone may require artwork + attachment confirmation + install note
- hardware-only item may not require artwork

Add a production readiness summary to:
- line item detail
- order detail
- supplier packet
- internal fulfillment command center

Make blocked items visually obvious.
