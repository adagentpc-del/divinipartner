Redesign the partner portal UI into a premium, luxury, high-conversion brand experience with selectable partner design templates.

Context:
The current partner portal looks too plain, bland, and generic. It only asks for colors, but the output does not feel branded, elevated, modern, or visually differentiated. The partner logo is also not displaying correctly.

Goal:
Create a more premium visual system where each partner portal can select from 2 to 3 luxury templates, upload or assign a logo, choose brand colors, and preview the portal design before publishing.

Important:
Do not break existing partner portal functionality.
Do not remove any existing partner, quote, document center, intake, upload, or admin features.
This is a UI and theme system upgrade only.
Use the existing data model where possible, but add fields as needed.
Keep all routes and flows working.

Primary design direction:
The partner portals should feel like premium branded microsites, not plain forms.

Visual inspiration:
1. Dark luxury portfolio style
2. Neon gradient creative studio style
3. Premium tech recruitment / corporate campaign style
4. Immersive event / experiential brand style
5. Sleek responsive website mockup style

The design should support:
- large partner logo
- premium hero section
- branded gradient or image background
- modern cards
- glassmorphism where appropriate
- soft shadows
- strong CTA buttons
- elevated typography
- mobile responsive layouts
- partner-specific color themes
- branded document/download sections
- branded quote/intake sections

Important implementation instructions:
1. Inspect the existing frontend styling system first.
2. Reuse Tailwind classes and existing component library if present.
3. Do not introduce heavy animation libraries unless already installed.
4. If animations are added, use CSS only or existing lightweight utilities.
5. Avoid increasing bundle size unnecessarily.
6. Keep the design editable and maintainable.
7. Do not use external image URLs.
8. Use uploaded images or CSS gradients.
9. Do not make all partners share the same theme unless no theme is saved.
10. Preserve existing partner portal data and flows.

Acceptance criteria:
- Partner logo displays correctly
- Admin can upload or replace partner logo
- Admin can choose from 3 portal templates
- Admin can customize colors and hero copy
- Admin can preview the design before publishing
- Public partner portal uses the published theme
- Customer portal looks premium and branded
- Intake, quote, upload, and document center flows still work
- Internal admin pages remain functional
- Mobile view looks polished
- No TypeScript or build errors
