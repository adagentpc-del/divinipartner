Now refine the existing Partner Commerce Portal by building a practical quote ingestion, catalog enrichment, and spec standardization workflow. The platform already has partners, portal types, products, packages, branding zones, quotes/spec references, financial scaffolding, inventory, reservations, fulfillment modes, and supplier routing. This pass should make it much easier to turn messy real world supplier information into structured internal catalog intelligence.

Important:
Do not rebuild from scratch.
Work from the existing codebase.
Preserve the current architecture, inventory logic, fulfillment modes, financial scaffolding, quote/spec references, and supplier routing.
Implement real workflow improvements, not placeholders only.

Primary objective for this pass:
Make the platform capable of ingesting quote files, screenshots, and manual supplier references, then converting them into cleaner structured backend product intelligence over time.
This should support a messy operational reality where the “catalog” may come from:
- supplier quotes
- ERP exports
- screenshots
- PDFs
- item photos
- vendor websites
- manual notes
- prior job references

This pass has 8 major priorities:

1. Quote ingestion workspace
2. Catalog enrichment workflow
3. Spec standardization model
4. Product creation from quote/source material
5. Internal review and approval workflow
6. Better mapping to suppliers, products, packages, and zones
7. Missing data and exception management
8. Cleaner internal intelligence views

-----------------------------------
1. QUOTE INGESTION WORKSPACE
-----------------------------------

Build a dedicated internal Quote Ingestion workspace for Super Admin and Internal Admin users.

This workspace should support ingesting and organizing source materials such as:
- PDF quotes
- PDF spec sheets
- images
- screenshots
- product photos
- supporting reference docs

For each uploaded source record, store:
- title
- supplier
- file type
- source type
  - quote
  - spec sheet
  - screenshot
  - website reference
  - ERP export
  - manual note
  - prior job reference
- upload file
- version
- effective date
- expiration date
- internal only flag
- source confidence / quality flag
- processing status
  - new
  - needs review
  - mapped
  - approved
  - archived
- notes

UI requirements:
- clean ingestion dashboard
- file cards or rows with thumbnails where applicable
- filters by supplier, source type, processing status, product mapping status
- quick preview drawer or modal
- bulk status updates if feasible
- easy “map to product” and “create product” actions

Do not make this feel like a raw document dump.
It should feel like an internal catalog intelligence workspace.

-----------------------------------
2. CATALOG ENRICHMENT WORKFLOW
-----------------------------------

Build a step by step enrichment workflow for turning source materials into structured product intelligence.

For each source record, allow internal admins to extract or fill in:
- display name
- internal name
- supplier name
- size / dimensions
- material
- finishing
- attachment method
- hardware included yes/no
- graphic only yes/no
- rental eligible yes/no
- reusable inventory compatible yes/no
- category
- lead time
- ops notes
- install notes
- customer facing summary
- backend ops summary

The workflow should support:
- creating a new product from a source
- linking the source to an existing product
- creating a new quote/spec reference attached to an existing product
- attaching the source to a package
- attaching the source to a branding zone

Build a guided enrichment panel, drawer, or wizard.
Do not rely on one giant ugly form.

- finishing summary
- attachment summary

- lead time

- install notes
- internal ops notes
- approved status

Create a clean UI for viewing these standard records on the product detail page.

-----------------------------------

Build exception handling for incomplete catalog intelligence.

Allow flagging:
- missing dimensions
- missing material
- missing attachment method
- missing lead time
- unclear hardware inclusion
- duplicate possible
- conflicting supplier info
- expired quote/spec
- missing customer facing summary
- missing backend ops summary

Create filters and badges for these missing data conditions.
Show them in the Quote Ingestion workspace and product detail pages.

This is important because the system will often be built from imperfect data.
The app should support incomplete but usable records while clearly showing what still needs cleanup.
