Build on our existing A3 partner portal system and upgrade it into a scalable, AI assisted, white labeled partner portal generator for A3 Visual.

Core objective

Create a reusable co branded partner portal system where each partner, such as Move Miami, Swarm, Wynwood Marketplace, and future partners, gets a branded client facing portal that feels like their brand first and A3 Visual second. The portal should allow clients to browse standard event products, venue specific branding opportunities pulled from a site survey and branding deck, branded event collateral, immersive upgrades, fabrication requests, and open ended custom event requests.

This must be built inside our existing system, not as a separate disconnected app.

Primary product goals

1. Fast partner setup
When creating a partner, I need to upload their logo, website, branding deck, and site survey or venue branding deck PDF. The system should use AI to extract colors and styling from the logo and website, suggest the partner theme, and extract venue branding locations and measurements from the uploaded deck. I then review and approve everything before it publishes.

2. White labeled co branded portals
Each partner portal should feel like the partner brand, using their visual style, colors, and approved branding, while clearly showing that A3 Visual is the production partner.

3. Two ordering paths
A. Standard standalone products where the client selects product, size, quantity, uploads artwork, and requests design if needed
B. Venue specific branding areas where the client selects an approved branded location from the site survey driven venue map and uploads artwork or requests design support. The client should not choose installation specs, mounting, lamination, grommets, or other technical production details for mapped venue locations because that is already handled by A3 from the site survey and backend data.

4. AI assisted deck ingestion
When a partner is created and I upload the venue branding deck PDF, the system should automatically analyze it, detect branding locations, measurements, and page references, then show me an admin review screen where I confirm, edit, delete, rename, merge, or approve the extracted locations before they go live in the portal.

5. Operational simplicity
For phase one, this is not full ecommerce checkout. This is a premium configured order request and quote workflow. Clients configure or select what they need, upload artwork, and submit. The system stores everything, notifies admin, and makes the request visible in the dashboard for review and quoting.

Design direction

The portal should look premium, polished, modern, and partner branded. It should not look like a cheap commodity print shop. It should feel like a strategic event branding portal.

Use a clean, luxury modern aesthetic with strong visual hierarchy, branded cards, section based modular design, strong buttons, image galleries, and configurable layout blocks.

Tech assumptions

Use the existing stack already in the project where possible.
Use Supabase for database and storage if the existing project already uses it.
Use server actions or API routes within the current Replit architecture.
Do not rebuild the whole app from scratch.
Extend the current codebase cleanly and modularly.
Create reusable components and admin workflows.
