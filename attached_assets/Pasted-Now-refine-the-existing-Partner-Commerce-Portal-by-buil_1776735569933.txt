Now refine the existing Partner Commerce Portal by building an automation and workflow orchestration layer. The platform already has partners, portal types, products, packages, branding zones, inventory, reservations, fulfillment modes, supplier routing, quote/spec references, quote ingestion/catalog enrichment, exports, reconciliation, communications, selective billing execution, and document/asset intelligence. This pass should make the platform proactively move work forward through rules, deadlines, task generation, and operational state changes.

Important:
Do not rebuild from scratch.
Work from the existing codebase.
Preserve the current architecture, role permissions, communications system, billing logic, supplier routing, and asset workflows.
Implement real orchestration improvements, not placeholders only.

Primary objective for this pass:
Make the platform operationally self-driving wherever possible.
Automatically create the right tasks, alerts, reminders, and status changes when orders, events, assets, invoices, and supplier assignments reach important workflow points.

This pass has 10 major priorities:

1. Workflow automation architecture
2. Rule-based trigger engine
3. Auto-created tasks and assignments
4. SLA and deadline monitoring
5. Automated blocked-state and readiness logic
6. Automated reminders and escalations
7. Workflow queue and orchestration dashboard
8. Better status transitions and guardrails
9. Audit history and override controls
10. Documentation and seed data

-----------------------------------
1. WORKFLOW AUTOMATION ARCHITECTURE
-----------------------------------

Build a practical automation/orchestration layer for the portal.

This should support:
- workflow triggers
- conditional actions
- task creation
- alert creation
- status updates
- communication draft generation
- escalation flags
- deadline monitoring

Do not build a giant abstract automation engine.
Keep it understandable and operational.

At minimum, create a workflow rule model that can support:
- trigger event
- optional conditions
- action(s)
- active/inactive
- applies to portal type(s)
- applies to role/object type
- notes

Supported object types should include:
- partner
- event
- order
- order line item
- supplier assignment
- invoice
- asset
- inventory reservation

-----------------------------------
2. RULE-BASED TRIGGER ENGINE
-----------------------------------

Build a first-pass trigger engine with sensible operational triggers.

Examples of triggers:
- order submitted
- order approved
- event created
- event approaching
- supplier assigned
- supplier status changed
- asset uploaded
- asset approved
- asset missing by deadline
- production blocked
- invoice created
- invoice sent
- payment overdue
- shortage detected
- inventory reservation shortage
- due date approaching
- reconciliation discrepancy flagged

For each trigger, support actions such as:
- create internal task
- create internal alert
- prepare communication draft
- update status
- flag blocked
- set escalation priority
- assign owner
- add timeline log entry

Do not overcomplicate rule building.
Use structured trigger/action definitions and clean admin UI.

3. AUTO-CREATED TASKS AND ASSIGNMENTS
-----------------------------------

Build lightweight operational tasks generated by workflow rules.

Each task should support:
- title
- linked object type/id
- task category
  - approval
  - asset follow-up
  - supplier follow-up
  - billing follow-up
  - reconciliation
  - inventory action
  - production review
  - event prep
- owner
- due date
- priority
- status
  - open
  - in progress
  - waiting
  - done
  - cancelled
- notes
- auto-created yes/no
- source workflow rule

Auto-create tasks for common situations such as:
- missing artwork after order submission
- supplier assignment still missing after approval
- shortage requires action
- proof approval needed
- invoice needs to be sent
- payment overdue follow-up
- commission discrepancy needs review
- event deadline approaching without readiness
- vendor issue flagged
- blocked line item needs resolution

Build:
- task list views
- task quick actions
- linked record navigation
- task panels on key detail pages

Keep this workflow-focused, not a full project management suite.

6. AUTOMATED REMINDERS AND ESCALATIONS
-----------------------------------

Build reminder and escalation workflows using the communications and alerts systems already in place.

Examples:
- remind client/partner to upload missing artwork
- remind internal admin that supplier is unassigned
- remind vendor that due date is approaching
- remind billing owner that invoice is overdue
- escalate blocked items that are unresolved within X days

- escalate discrepancy records awaiting reconciliation

Requirements:

- assign follow-up tasks where appropriate

- support escalation priority levels:
  - low
  - medium
  - high
  - urgent

Do not spam everything.
Make rules understandable and useful.

8. BETTER STATUS TRANSITIONS AND GUARDRAILS
-----------------------------------

Build safer and smarter status transition logic.

Examples:
- prevent line item from being marked production ready if required assets are missing
- warn before marking vendor complete if shipment/delivery status is missing
- warn before approving order if supplier assignment is incomplete
- warn before vendor release if artwork is not approved
- warn before event marked ready if blocked items exist

Requirements:
- clear validation messages
- override option for internal admin with required note
- timeline log entry for overrides
- visible badges when a record was manually overridden

This should prevent operational mistakes without becoming rigid and annoying.

Track:
- workflow-triggered actions
- task creation history
- status auto-updates
- alert creation
- communication draft creation
- blocked/unblocked changes
- manual overrides
- who made the change
- when
